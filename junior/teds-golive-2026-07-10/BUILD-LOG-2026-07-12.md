## Item 1 â€” Brand inheritance into the HealthPilot quiz embed

Status: DONE (sending side + full contract doc for the receiving side).

What changed:
- `frontend/src/components/health-quiz/HealthPilotEmbed.tsx`
  - New exported pure helper `buildEmbedTheme(clinic)` that maps the sandbox
    `Clinic` brand tokens onto HealthPilot's `EmbedTheme` and is forwarded on
    every `hp:init` (in reply to `hp:ready`). Every field is conditional so a
    half-configured clinic never overwrites a HealthPilot default.
  - New exported `fontFamilyToFontKey()` â€” HealthPilot only accepts a closed
    set of CSP-safe font *register keys* (`sans|grotesk|serif|editorial|mono`),
    never a raw family string, so the clinic's stored Google-Fonts `fontFamily`
    is mapped to the nearest register (serif families -> `serif`, mono ->
    `mono`, else `sans`; absent -> undefined so HP keeps its default).
  - `sendInit` now sends `theme: buildEmbedTheme(clinic)` instead of the old
    inline `{ primaryColor, logoUrl, brandName }`.
- `frontend/src/components/health-quiz/HealthPilotEmbed.test.tsx`
  - +3 describe blocks / new cases: `fontFamilyToFontKey` mapping,
    `buildEmbedTheme` full/partial/data-URI, and a full-clinic `hp:init`
    forwarding integration test.

Token map (sandbox Clinic -> HealthPilot EmbedTheme):
| Clinic field       | EmbedTheme field | Notes |
|--------------------|------------------|-------|
| primaryColor       | primaryColor     | hex, receiver re-sanitises |
| accentColor        | accentColor      | hex |
| backgroundColor    | surfaceColor     | hex (standard preset only) |
| textColor          | inkColor         | hex (standard preset only) |
| fontFamily         | fontKey          | mapped to a register key |
| logoUrl            | logoUrl          | made absolute; data: kept verbatim |
| name               | brandName        | text |
| welcomeMessage     | tagline          | text, receiver length-clamps |

Not forwarded, with reason:
- `buttonColor` â€” HealthPilot's EmbedTheme has no button slot; it derives CTA
  colour from primary/accent. Left out rather than invent a receiver field.
- `radius` â€” the sandbox `Clinic` model has NO radius/corner token (confirmed
  in `clinic-brand.tsx` `ClinicBrandConfig`). HealthPilot accepts a `radiusKey`
  (`sharp|subtle|soft|round`). FOR FELIX: if a per-clinic radius is wanted, add
  a `radiusKey`-style column to `Clinic` + expose it in `ClinicBrandConfig`,
  then add `radiusKey` to `buildEmbedTheme`. One-line change here once the field
  exists.
