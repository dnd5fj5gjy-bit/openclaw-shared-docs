# Nightly Felix Finishable Output Playbook

## Purpose
When a late-night prompt asks for a useful surprise, default to something that is:
- immediately reusable
- small enough to finish in one pass
- valuable without additional context
- specific to Felix's actual operating style

## Best default outputs
1. **Internal playbook/template** — fastest to ship, highest reuse.
2. **Tiny utility** — only if it solves one annoying repeated task.
3. **Concise brief** — only if it changes a decision or clarifies a process.

## Selection rule
Choose the output that best satisfies all three:
- **Finishable tonight**: no dependencies, no waiting, no external approvals.
- **High leverage**: saves time for Felix or the team repeatedly.
- **Low ceremony**: can be read in under 3 minutes.

## What to avoid
- vague brainstorming
- large multi-step projects
- anything that needs a follow-up question
- generic productivity advice

## Strong candidate topics for Felix
- decision templates for BGV ops
- launch checklist for small web tools
- incident triage checklist
- teammate handoff format
- quick content QA checklist
- nightly “what matters tomorrow” review

## Recommended default format
Use this structure:

### Title
Short, useful, obvious.

### When to use
One sentence.

### The template/playbook
Bullet points or a fill-in-the-blank format.

### Good example
One realistic example.

### Why this helps
One sentence.

## Success metric
If Felix can use it without asking for clarification, it counts.

## If unsure
Prefer the template. It is the most reusable surprise output.
