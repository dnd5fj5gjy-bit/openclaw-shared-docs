2 · Inter-agent communication channels

agent-comms.jsonl Broadcast log
~/agents/nexus/workspace/agent-comms.jsonl
All agents append JSON lines here. Used for status updates, insights, ideas. Never carries user-private context (post-isolation rule, 2026-05-02). Format: {id, from, to, message, type, timestamp}. Types: insight, idea, status, question, task, message.
agent-tasks.jsonl Task tracking
~/agents/nexus/workspace/agent-tasks.jsonl
Active/queued/done state for cross-agent work. Format: {id, agent, title, status, startedAt, ...}.
pending-messages.jsonl Direct messages
~/agents/<agent>/workspace/pending-messages.jsonl
Per-agent DM inbox. Used for agent-to-agent direct messages and listener-injected user messages. Drained by the receiving agent's heartbeat or listener.
share-knowledge.py Formal hand-off
~/agents/shared/tools/share-knowledge.py
Sole sanctioned path to copy a piece of knowledge from one agent to another. Logs provenance (who shared, when, why). Used after isolation lockdown.
exportable.md Whitelist (per-agent)
~/agents/<agent>/self-improving/exportable.md
The ONLY part of an agent's self-improving memory that other agents may ingest. Contains operational lessons stripped of user-private context. Q's ingestion script reads only this file from each peer.
self-improving/ (PRIVATE) Walled garden
~/agents/<agent>/self-improving/
Private memory. No other agent reads. Holds memory.md (HOT), corrections.md, personality.md, projects/, domains/, people/, archive/. Cross-agent access goes through the four channels above — not by reading each other's files.
