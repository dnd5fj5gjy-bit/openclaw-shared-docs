Implementation Notes

Before going live: Update all https://tedshealth.com references in the pages to match the actual production domain. The generator uses this as a constant — a single find/replace handles it.

Option 1 — CMS (recommended)

Copy body content from each file into the CMS. The HTML files are self-contained but designed to slot into the existing Ted's Health template. Extract the content between <div class="hero"> and the closing </div class="content"> — that's the page-specific content. Meta tags, schema, breadcrumb, nav, and footer are already in the CMS template.

Create pages at the URLs listed above. Paste the extracted content. Add meta title and description from the <head>. Paste both JSON-LD blocks into the schema/head injection field of the CMS.

Option 2 — Static deploy

The files are fully self-contained HTML — header, nav, footer, styles all inline. Deploy to any static host (Netlify, Vercel, GitHub Pages, S3). Create the directory structure to match the target URLs:

trt-symptoms/
├── index.html          → /trt-symptoms/
├── brain-fog/
│   └── index.html      → /trt-symptoms/brain-fog/
├── low-energy/
│   └── index.html      → /trt-symptoms/low-energy/
... (etc)

Rename each [slug].html to [slug]/index.html to get clean URLs without .html extensions.

Option 3 — Quick preview (immediate)

Open any HTML file directly in a browser. No server needed — they're fully static. Works offline.

Re-generating pages: The generator script at workspace/tools/generate_symptom_pages.py produces all 10 pages. To update copy or add keywords: edit the SYMPTOMS list in the script and re-run python3 workspace/tools/generate_symptom_pages.py. All pages regenerate in under 2 seconds.
