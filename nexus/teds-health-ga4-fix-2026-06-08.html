Ted's Health — GA4 Cross-Domain Fix

Nexus  Risk #1 — Score 20/25   June 8, 2026  ·  For: Ted's Health developer

Why this matters urgently: Every time a patient moves from tedshealth.com to connect.tedshealth.com (the signup flow), Google Analytics loses the session. From GA4's perspective, this looks like 100% bounce rate on paid campaigns and zero attributed conversions. If Ted's is running Google Ads or Meta, the platforms are bidding blind — they have no conversion signal to optimise against. This is not a minor tracking gap. It is the difference between knowing whether marketing works and guessing.

The Problem — Exactly What's Happening

GA4 uses a first-party cookie tied to the domain (_ga, _ga_XXXXXXXX). When a user lands on tedshealth.com, GA4 writes those cookies with domain=tedshealth.com. The moment they navigate to connect.tedshealth.com, the browser cannot read those cookies — it's a different origin. GA4 creates a brand-new session marked as "(direct)" traffic with no referral, effectively hiding where the user came from.

The fix is cross-domain measurement: GA4 appends a URL parameter (_gl) to links crossing the domain boundary. The receiving page reads that parameter and merges the session. The result: one continuous session from first click to conversion, with the original source (Google Ads, Meta, organic) intact.

Before fix

User from Google Ad → tedshealth.com (session: source=cpc/google) → connect.tedshealth.com (new session: source=direct). Conversion attributed to "(direct)".

After fix

User from Google Ad → tedshealth.com → connect.tedshealth.com (same session continues). Conversion attributed to "cpc / google". Google Ads gets the signal.

Implementation — Step by Step

1

Find your GA4 Measurement ID

2 mins

In GA4: Admin → Data Streams → your web stream. Copy the Measurement ID (format: G-XXXXXXXXXX).

You will need this in every step below. Write it down now.

2

Configure cross-domain in GA4 Admin

5 mins

GA4 Admin → Data Streams → your stream → Configure tag settings → Configure your domains.

Add both domains:

Domain 1tedshealth.com
Domain 2connect.tedshealth.com
Match typeContains

Save. This tells GA4 that navigation between these two domains should be treated as one session.

3

Option A: Via Google Tag Manager (preferred)

20 mins

3a — Create the GA4 Configuration Tag

In GTM: Tags → New → Google Tag. Set Tag ID to your G-XXXXXXXXXX.

Under Configuration Settings → Cross-domain linking, add:

Accept incomingEnabled (checked)
Decorate formsEnabled (checked)
Decorate linksEnabled (checked)
Domainstedshealth.com, connect.tedshealth.com

Trigger: All Pages.

3b — Deploy to both properties

If the main site (tedshealth.com) and the signup app (connect.tedshealth.com) are separate codebases, GTM must be installed on both. The cross-domain parameter only works end-to-end when both domains have the same GA4 tag running.

Confirm GTM is present on connect.tedshealth.com by opening the signup page, opening DevTools → Network, filtering for googletagmanager. If you don't see it — install GTM on the app before proceeding.

4

Option B: Direct gtag.js implementation (no GTM)

15 mins

If not using GTM, add this to the <head> of both tedshealth.com and connect.tedshealth.com:

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());

  gtag('config', 'G-XXXXXXXXXX', {
    // Cross-domain: treat these two domains as one property
    'linker': {
      'domains': ['tedshealth.com', 'connect.tedshealth.com'],
      'decorate_forms': true,
      'accept_incoming': true
    }
  });
</script>

Important: both sites must use the same G-XXXXXXXXXX ID for the cross-domain linker to work.

5

Track the signup conversion event

30 mins

Cross-domain tracking preserves the session, but you also need to fire a conversion event when a patient completes signup. This tells GA4 (and Google Ads) that a conversion happened.

Option A: Fire from the app on successful signup

// In connect.tedshealth.com app — fire after successful account creation
gtag('event', 'sign_up', {
  method: 'email',
  // Add any custom dimensions useful for reporting
  patient_type: 'new',
  source_form: 'trt_consultation_request'
});

// Also fire 'generate_lead' for Google Ads optimisation
gtag('event', 'generate_lead', {
  value: 150,       // estimated LTV / consultation value in GBP
  currency: 'GBP'
});

Option B: Via GTM trigger on signup confirmation URL

If the signup app redirects to a thank-you URL on completion (e.g. /signup/complete), create a GTM trigger:

Trigger typePage View
Fires onPage URL contains /signup/complete (or whatever the success URL is)
TagGA4 Event — event name: sign_up

Then mark sign_up as a conversion in GA4 Admin → Events → mark as conversion.

6

Link GA4 to Google Ads

10 mins

If Ted's is running Google Ads, link the two so conversion data flows back to the bidding system:

  • GA4 Admin → Product Links → Google Ads Links → Link
  • Select the Google Ads account
  • Enable "Enable Personalised Advertising" and "Import GA4 conversions"
  • In Google Ads: Tools → Conversions → import from GA4 → select sign_up

Once linked, Google's Smart Bidding can optimise for actual signups rather than clicks.

Verification Checklist

After implementing, verify everything is working before declaring done.

  • Open tedshealth.com. Check Network tab for requests to analytics.google.com. You should see a hit with tid=G-XXXXXXXXXX.
  • Click any link that goes to connect.tedshealth.com. The URL should have a _gl= parameter appended automatically. This confirms the linker is working.
  • Open GA4 → Realtime report. Navigate from main site to signup app. You should see ONE user, not two separate sessions.
  • Check that the sign_up event fires on successful registration. Visible in GA4 Realtime → Events.
  • If using Google Ads: after 48 hours, conversions should appear in Google Ads campaign reports.
  • In GA4 → Reports → Acquisition → Traffic Acquisition: "Organic Search" and "Paid Search" should now show conversions, not just sessions.

Google Search Console — Precondition for SEO Measurement

None of the SEO monitoring above is possible without Search Console registered. It is a 10-minute task with no developer required. Once verified, it unlocks: keyword positions, click data, crawl errors, indexing status, Core Web Vitals. Without it, Ted's Health has no way to know what's ranking or why organic traffic changes.

  • Go to search.google.com/search-console
  • Add property → URL prefix → enter https://www.tedshealth.com
  • Verify via Google Tag Manager (recommended — no DNS or file upload needed)
  • Submit sitemap: enter https://www.tedshealth.com/sitemap.xml
  • Wait 48–72 hours for initial data to appear
